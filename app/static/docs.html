/api/summarize

Input: Text to summarize and length of summary as JSON
Output: Summary in JSON

Input is provided via POST request. Input data is serialized as JSON, this is done both for practical reason and for convenience. The practical reason being it's easy to encode and decode JSON both in Javascript and Python still being in an easy to understand format. The general format is as given:


    {
        params: source_text,
    }

Output is returned as a result of POST request. Again, JSON format is used and format is as given:


    {
        [index, line]
         ..........
    }

where index is the location of the line within the source text and line is simply the generated summary.
